// Key data flows
Article publish flow
01Write in Notion Articles DB → set Status = Ready, check SEO Checklist
02Vercel Cron fires every 30 min → calls POST /api/sync/notion
03notion-sync.ts validates fields, converts blocks, uploads hero image to Sanity CDN
04createOrReplace into Sanity with stable ID notion-{pageId}
05Notion Status promoted Ready → Published automatically
06Next.js ISR revalidates /articles/[slug] within 60s
Skills Hub update flow
01Manually create skillSource record in Sanity Studio (status: published)
02OR: Community submits via /skills/submit → lands in Notion Skill Submissions DB as Pending
03Editorial review in Notion → set Status = Approved
04Firecrawl weekly cron crawls all published sources, updates skillCount + skillCountPrev diff
05/skills page revalidates hourly, delta badge appears on cards where count changed
Schema enforcement layers
L1Human gate: Notion SEO Checklist checkbox + Status must be Ready or Published to enter sync
L2Sync script gate: validates required fields (title, slug, author, category) — skips + logs if missing
L3Sanity required() validators: reject malformed writes at the CMS layer
—No manual policing needed. All three layers are automatic.
