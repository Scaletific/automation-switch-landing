SKILL.md Files: The Configuration Primitive Every Agent Builder Should Know

M
Michael Nouriel
Founder, Automation Switch
29 March 2026
12 min read
Deep Dive
Key Takeaways
  • A SKILL.md file is a markdown document that gives an AI coding agent a focused operating mode for a specific task or domain.
  • SKILL.md files live in a skills directory inside your project and are loaded by the agent when invoked by name or trigger pattern.
  • The community has built hundreds of SKILL.md files covering engineering, DevOps, product, marketing, and data domains.
  • The configuration pattern works across Claude Code, Cursor, Copilot, and most major AI coding agents.
  • Writing effective SKILL.md files requires specifying constraints, context, and decision patterns — not just instructions.

The Problem With General-Purpose Agents

Every AI coding agent ships as a generalist. It knows how to write Python, debug TypeScript, explain architecture decisions, and draft commit messages. That breadth is the product. But breadth is also the problem.

When you ask a general-purpose agent to review a pull request, it has no idea what a good PR looks like in your codebase. It does not know your test coverage standards, your naming conventions, your tolerance for comments in code, or your preference for atomic commits. It will give you a review that would apply to any codebase, which means it applies fully to none.

SKILL.md files solve this by giving the agent a focused operating mode before it starts work.

What a SKILL.md File Actually Is

A SKILL.md file is a markdown document — typically 100 to 400 lines — that describes how an agent should behave within a specific domain or task. It is not a prompt. It is a configuration document. The distinction matters.

# code-review SKILL

## When to activate
When invoked with: review, pr-review, code review

## Context
You are reviewing code in a Next.js 15 App Router project.
TypeScript strict mode. No class components. Server components by default.

## What to check
- Type safety (no implicit any, no type assertions without justification)
- Missing error boundaries on client component trees
- Unused imports and dead code paths
- N+1 query patterns in server components...

The agent reads this file before it starts work. The result is a reviewer that knows your stack, your standards, and your tolerance for pedantry — not a generic assistant applying universal best practices.

Where SKILL.md files live

By convention, SKILL.md files live in a .claude/skills/ directory at the root of your project, alongside your CLAUDE.md working memory file. Each skill is its own subdirectory containing the SKILL.md and any supporting files the skill needs.

The naming convention is intentional: SKILL.md mirrors the structure and purpose of CLAUDE.md, but scoped to a domain rather than a project.

The Community Has Already Done the Hard Work

There are now several hundred community-contributed SKILL.md files indexed across public repositories. Engineering skills covering code review, debugging, architecture decisions, and testing strategy. DevOps skills for deployment checklists and incident response. Product skills for sprint planning and stakeholder updates. Marketing skills for brand voice and content creation.

Most of these can be used directly or adapted in under ten minutes. The barrier to a well-configured agent is lower than almost anyone realises.

FAQ
Do SKILL.md files work with Cursor and Copilot, or only Claude Code?
The SKILL.md pattern originated in the Claude Code ecosystem but has been adopted across Cursor, Copilot, and other AI coding agents. The file format is plain markdown, so the same file works across platforms with minimal adjustment to invocation syntax.
How long should a SKILL.md file be?
Effective SKILL.md files are typically 100 to 400 lines. Shorter than that and the skill lacks enough context to meaningfully constrain agent behaviour. Longer than that and you risk diluting focus. If a skill is growing past 500 lines, it is usually covering two domains and should be split.
Where is the best place to find existing SKILL.md files?
The Automation Switch skills directory indexes the most active community repositories and official platform collections. Each source is crawled weekly and the skill count is kept current, so you can see which sources are actively maintained.

About This Source

The Claude Code Skills Hub is the canonical source for SKILL.md files built and maintained by Anthropic. Each skill in the collection is designed to work with Claude Code's plugin and skills system, and follows the SKILL.md specification developed by the Claude team.

Skills are organised by domain and subdomain. Each domain folder contains a SKILL.md with detailed agent instructions, supported by reference files where needed. The collection is updated whenever Anthropic ships new capability areas for Claude Code.
